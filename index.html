equity‑analyst -- AI‑powered equity research platform

An end-to-end automated system that analyses ASX and US stocks every morning, generates ML-scored trade signals, and delivers prioritised research reports with specific entry, stop, and target levels -- before market open.

The hardest part of building AI systems that make consequential decisions is not the AI. It is knowing when to trust it, when to override it, and how to build the measurement infrastructure that tells you which is which. This project exists to answer those questions with data rather than opinion -- and to keep asking them every day the system runs.

Most AI systems in finance optimise for signal generation. This one optimises for decision quality -- by treating measurement, validation, and honest reporting as first-class engineering concerns, not afterthoughts. Every architectural choice below reflects that priority.

Engineering decisions worth noting

Champion/Challenger -- not version-over-time

Comparing strategy v1 (Jan-Jun) vs v2 (Jun-Dec) conflates strategy quality with market regime. Both see the same candidates on the same day. The comparison signal lives only in divergences.

Decision: apples-to-apples beats historical comparison
Promotion policy decoupled from engine

Changing promotion criteria (win rate gap vs R-multiple vs statistical significance) should not require touching the comparison engine. The policy is a swappable Protocol class. Four implementations ship; the default requires both win rate AND avg R to improve.

Decision: interface-first, tight coupling is a maintenance trap
Bull/Bear debate -- forced adversarial reasoning

A single model asked "should I take this trade?" finds reasons to say yes. Separate Bull and Bear instances with assigned stances surface the bear case confirmation bias would suppress. Debate log tracks whether SKIP verdicts actually avoided losers.

Decision: adversarial prompting over single-pass optimism
Validation speed as a strategic constraint

A walk-forward validation run taking 150 minutes isn't just slow -- it stops being used. Strategy decisions get made on intuition when the feedback loop is too long to run regularly. The bottleneck was traced and eliminated at the right layer, reducing validation time from hours to minutes without compromising the correctness guarantees.

Decision: a feedback loop only exists if people use it. Speed is a correctness requirement, not a performance optimisation.
Validation integrity over vanity metrics

Early validation showed strong results. A systematic review found the methodology was overstating performance -- the validation harness was not testing the signals actually being traded, and feature construction introduced look-ahead bias. The honest number was materially lower. The methodology was corrected, the harness tightened, and go-live blocked until the evidence supported it.

Decision: surface the honest number, fix the methodology, delay go-live until the evidence supports it. A system that tells you the truth when results are bad is worth more than one that only tells you the truth when they're good.
Architectural memory in the repo, not in someone's head

CI auto-fixes test failures via Claude Code. But an automated agent that makes tests pass without understanding why an invariant exists will make the wrong fix -- weakening a correctness guarantee instead of fixing the test setup. Architectural decisions and their reasoning are encoded in CLAUDE.md, which the agent reads before touching anything.

Decision: institutional memory belongs in the repo. Autonomous tooling is only as good as the context it operates in.

Built for reliability

Each stage is independent and fails gracefully. A slow API call in one step never blocks the rest. The pipeline always produces a report.

The measurement stage is what separates this from a signal screener. Every decision is logged, scored against what actually happened, and fed back into the evaluation loop. Strategy changes are judged on data, not feel.

How Claude is used

  • Claude Haiku -- stock analyst. Each morning, 48 stocks are analysed with signals, ML scores, news, and macro context injected into the prompt. Haiku produces a structured JSON verdict: watch level, confidence, key signals, and summary.
  • Claude Sonnet (Bull) -- argues forcefully FOR each candidate trade, citing signals, momentum, and sector tailwinds.
  • Claude Sonnet (Bear) -- argues forcefully AGAINST, citing headwinds, walk-forward reliability, and false-positive risk.
  • Claude Sonnet (Judge) -- weighs both cases and returns TAKE / SKIP / REDUCE_SIZE with reasoning. Only candidates that survive the debate reach the portfolio manager.
  • Claude Sonnet (Portfolio Manager) -- ranks the day's candidates into actionable decisions: ENTER, CLOSE, TRAIL_STOP, WATCH, SKIP. CGT-aware (flags positions within 60 days of the 12-month discount). Debate verdicts are included as second opinions.
Why multiple agents?

A single model asked "should I take this trade?" will find reasons to say yes. Forcing separate Bull and Bear instances to argue opposite sides surfaces the bear case that confirmation bias would suppress.

The Judge then evaluates both arguments with a clear decision rule: when genuinely uncertain, SKIP. Missing a trade costs nothing; a bad trade costs real money. The debate log tracks whether SKIP verdicts actually avoided losers.

Models: claude-haiku-4-5 (analysis) + claude-sonnet-4-20250514 (debate + PM)

Walk-forward validation (3 years)

Each strategy is validated using an expanding-window walk-forward methodology across 3 years of price history. A 10-day embargo gap between train and test prevents forward-return labels overlapping the test window (Lopez de Prado 2018, §7.4).

  • Robust -- signal fires consistently, train and validation win rates both strong. These tickers are eligible for paper trading.
  • Moderate -- some signal reliability; not traded, kept on watchlist for monitoring.
  • Overfit -- high train win rate, low validation win rate. Excluded from trading entirely.
