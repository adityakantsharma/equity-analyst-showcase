How Claude is used

  • Each stock gets a structured prompt with price action, 50+ signal states, ML ensemble score, macro context (RBA rate, VIX, yield curve, AUD/USD), and recent news headlines
  • Claude returns a confidence score (1–5), watch level (acting / interesting / monitor), key catalysts, risk factors, and a specific trade idea with entry, stop, and target
  • Fully agentic — no human in the loop. 47 stocks analysed, report generated, and email sent in ~35 minutes
  • Upcoming earnings dates and EPS results are injected into the prompt automatically, giving Claude context on near-term catalysts

What I learned: Building a production ML trading system end-to-end — from raw price data to actionable signals — requires as much work on data quality, signal validation, and walk-forward testing as on the models themselves. The highest-edge strategy (73% win rate) uses a small set of confirmed signals, not the full ML feature set. Claude acts as the synthesis layer — turning structured signal data into a coherent trade thesis with specific levels. Adding MLflow and FastAPI transformed the project from a batch script into a deployable ML service.
