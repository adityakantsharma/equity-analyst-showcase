Daily pipeline -- runs at 7:30am via cron

  1. Price fetch -- yfinance OHLCV for all 48 tickers
  2. News fetch -- yfinance + Google News RSS (no API key, unlimited)
  3. AI analysis -- Claude Haiku per stock, signals + ML score injected into prompt
  4. Ensemble ML scoring -- RF + LightGBM + Panel, walk-forward validated
  5. Bull/Bear debate -- Claude Sonnet argues for and against each candidate; judge returns TAKE/SKIP/REDUCE_SIZE
  6. Champion/Challenger -- both v1.0 and v2.0 strategies decide on identical candidates; divergences logged for comparison
  7. Portfolio Manager -- Claude Sonnet ranks actions with debate verdicts + CGT awareness
  8. Report generation -- HTML + JSON daily report with sector heatmap, ML scores, paper trades
  9. Outcome backfill -- scores debate + strategy logs once their 10-day horizon matures
  10. Drift check + MLflow logging -- detects model decay, promotes if AUC improves
  11. Alerts + email -- morning email with PM recommendation + system metrics panel
  12. Fridays: walk-forward validation (48 tickers, 3yr history, 10-day embargo gap)
  13. Fridays: signal accuracy cache refresh -- pre-live gate status updated
Why 13 steps?

Each step is independent and fails gracefully. If the debate agent hits a rate limit, the PM still runs without verdicts. If backfill finds no mature decisions, it exits cleanly. The pipeline always produces a report.

The measurement steps (6, 9, 13) are what separate this from a signal screener. Every decision is logged, scored, and compared. Strategy changes are evaluated on data, not feel.

How Claude is used

  • Claude Haiku -- stock analyst. Each morning, 48 stocks are analysed with signals, ML scores, news, and macro context injected into the prompt. Haiku produces a structured JSON verdict: watch level, confidence, key signals, and summary.
  • Claude Sonnet (Bull) -- argues forcefully FOR each candidate trade, citing signals, momentum, and sector tailwinds.
  • Claude Sonnet (Bear) -- argues forcefully AGAINST, citing headwinds, walk-forward reliability, and false-positive risk.
  • Claude Sonnet (Judge) -- weighs both cases and returns TAKE / SKIP / REDUCE_SIZE with reasoning. Only candidates that survive the debate reach the portfolio manager.
  • Claude Sonnet (Portfolio Manager) -- ranks the day's candidates into actionable decisions: ENTER, CLOSE, TRAIL_STOP, WATCH, SKIP. CGT-aware (flags positions within 60 days of the 12-month discount). Debate verdicts are included as second opinions.
Why multiple agents?

A single model asked "should I take this trade?" will find reasons to say yes. Forcing separate Bull and Bear instances to argue opposite sides surfaces the bear case that confirmation bias would suppress.

The Judge then evaluates both arguments with a clear decision rule: when genuinely uncertain, SKIP. Missing a trade costs nothing; a bad trade costs real money. The debate log tracks whether SKIP verdicts actually avoided losers.

Models: claude-haiku-4-5 (analysis) + claude-sonnet-4-20250514 (debate + PM)

Walk-forward validation (3 years)

Each strategy is validated using an expanding-window walk-forward methodology across 3 years of price history. A 10-day embargo gap between train and test prevents forward-return labels overlapping the test window (Lopez de Prado 2018, §7.4).

  • Robust -- signal fires consistently, train and validation win rates both strong. These tickers are eligible for paper trading.
  • Moderate -- some signal reliability; not traded, kept on watchlist for monitoring.
  • Overfit -- high train win rate, low validation win rate. Excluded from trading entirely.

Engineering decisions worth noting

Champion/Challenger -- not version-over-time

Comparing strategy v1 (Jan-Jun) vs v2 (Jun-Dec) conflates strategy quality with market regime. Both see the same candidates on the same day. The comparison signal lives only in divergences.

Decision: apples-to-apples beats historical comparison
Promotion policy decoupled from engine

Changing promotion criteria (win rate gap vs R-multiple vs statistical significance) should not require touching the comparison engine. The policy is a swappable Protocol class. Four implementations ship; the default requires both win rate AND avg R to improve.

Decision: interface-first, tight coupling is a maintenance trap
Bull/Bear debate -- forced adversarial reasoning

A single model asked "should I take this trade?" finds reasons to say yes. Separate Bull and Bear instances with assigned stances surface the bear case confirmation bias would suppress. Debate log tracks whether SKIP verdicts actually avoided losers.

Decision: adversarial prompting over single-pass optimism
NewsAPI replaced with yfinance + Google News RSS

The 100 req/day free tier was exhausted by a 48-ticker watchlist. The next tier was $449/month. yfinance .news + Google News RSS covers the same signal for free, with no rate limits and no API key. News is context for the analyst, not the primary signal.

Decision: right-size dependencies, cost scales with value

What I learned: Building a production ML trading system end-to-end -- from raw price data to actionable signals -- requires as much work on data quality, signal validation, and walk-forward testing as on the models themselves. The highest-edge strategy uses a small set of confirmed signals, not the full ML feature set. Claude acts as the synthesis layer, turning structured signal data into a coherent trade thesis with specific levels. Adding MLflow and FastAPI transformed the project from a batch script into a deployable ML service.
